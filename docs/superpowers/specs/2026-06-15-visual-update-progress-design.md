# 可视化更新进度设计

## 目标

为 Windows 文件整理助手增加完整、可见且安全的更新体验：

- 下载阶段显示进度、已下载大小、平均速度和预计剩余时间。
- 下载速度每秒刷新一次，只显示从下载开始计算的平均速度。
- 下载和校验阶段允许用户停止更新。
- 一旦进入程序文件替换阶段，禁止关闭和停止，避免安装损坏。
- 更新期间主窗口整体禁用，只允许操作更新窗口。
- 自动检查和手动检查发现新版后统一进入同一套可视化流程。
- 独立 `updater.exe` 显示备份、安装、完成、回滚状态。

## 视觉方向

界面采用“精密工具面板”风格，与现有应用保持一致：

- 主色使用现有深蓝黑 `#101820`、橙色 `#F05A28` 和浅灰背景 `#EEF2F4`。
- 成功状态使用绿色 `#176342`，错误状态使用深红 `#9B3417`。
- 更新窗口保持克制、清晰和高对比，不使用渐变、玻璃拟态或装饰性动画。
- 信息层级依次为：当前阶段、进度条、核心数字、辅助说明、操作按钮。
- 使用微软雅黑 UI，避免引入额外字体文件和打包依赖。
- 状态切换使用短暂淡入或颜色变化；下载数字每秒更新，不持续闪烁。

## 主程序更新窗口

更新窗口尺寸约为 `560x500`，只允许存在一个实例。窗口由以下区域组成：

1. 顶部标题区
   - 标题：`软件更新`
   - 显示当前版本和目标版本。
   - 右上角不提供额外关闭按钮；系统关闭按钮由状态机控制。

2. 阶段指示区
   - 四个阶段：`下载更新`、`校验文件`、`准备安装`、`安装更新`。
   - 当前阶段使用橙色强调，已完成阶段使用绿色，未开始阶段使用灰色。

3. 进度区
   - 确定总大小时使用确定进度条和百分比。
   - 服务器没有提供 `Content-Length` 时使用不确定进度条，仍显示已下载大小和平均速度。
   - 下载完成后进度条保持 100%，状态切换为校验。

4. 数据区
   - 已下载：`12.4 MB / 20.8 MB`
   - 平均速度：`2.1 MB/s`
   - 预计剩余：`约 4 秒`
   - 数据每秒刷新一次。
   - 平均速度计算方式为累计下载字节数除以累计有效下载时间。

5. 信息区
   - 显示当前动作和不可逆边界说明。
   - 错误时显示可理解的中文原因，不直接只显示异常类型。

6. 操作区
   - 下载和校验阶段：`停止更新`
   - 停止后：`重新开始更新`
   - 发现新版但尚未下载：`立即更新`
   - 准备安装及安装阶段：按钮禁用并显示 `正在安装…`

## 状态机

主程序更新状态：

```text
available
  -> downloading
  -> verifying
  -> preparing_install
  -> updater_started

downloading
  -> cancelled
  -> failed

verifying
  -> cancelled
  -> failed

cancelled
  -> downloading

failed
  -> downloading
  -> checking
```

规则：

- `downloading` 和 `verifying` 可停止。
- `preparing_install` 开始后取消按钮立即禁用。
- `updater_started` 后主程序关闭。
- 用户停止后删除未完成 ZIP，解除操作锁，从 0% 重新下载。
- 不支持断点续传。

## 下载进度与取消

`update_manager.py` 提供带回调和取消令牌的下载接口：

- 每读取一个数据块，报告累计字节数和响应头中的总字节数。
- GUI 只每秒刷新一次速度和剩余时间，避免过度更新 Tk 控件。
- 取消令牌使用 `threading.Event`。
- 下载循环和 SHA-256 校验循环都检查取消令牌。
- 取消时关闭响应、删除临时 ZIP，并抛出专用的“用户取消”结果。
- 校验失败同样删除 ZIP。

进度事件包含：

- `phase`
- `downloaded_bytes`
- `total_bytes`
- `elapsed_seconds`
- `average_bytes_per_second`
- `estimated_seconds_remaining`

## 主窗口锁定

用户点击“立即更新”并成功取得更新操作锁后：

- 主窗口覆盖一个半透明深色遮罩。
- 遮罩显示 `软件正在更新，请在更新窗口中操作`。
- 主窗口中的路径、模式、配置、预览和运行按钮全部不可操作。
- 更新窗口保持在主窗口上方。
- 下载停止或失败后移除遮罩并释放 `OperationGate`。
- 下载完成进入安装阶段后保持遮罩，直到主程序关闭。
- 更新过程中点击主窗口关闭按钮不退出程序，而是将更新窗口提升到前台。

整理任务已经运行时不能开始更新；更新已经运行时不能开始整理任务。

## 自动检查与手动检查统一

- 自动检查发现新版后打开同一个更新窗口，不再使用独立确认弹窗。
- 手动检查发现新版也进入相同的 `available` 状态。
- 两条入口共用同一个下载、取消、校验和安装流程。
- 已是最新版、检查失败和重新检查继续在该窗口中显示。

## 独立更新器窗口

`updater.exe` 从无界面的后台程序改为小型 CustomTkinter/Tkinter 安装窗口，尺寸约为 `480x320`。

阶段：

1. `等待主程序退出`
2. `正在备份现有程序`
3. `正在安装新版本`
4. `正在完成更新`
5. `更新完成，即将重新启动`

安装器行为：

- 进入安装器后不允许取消或关闭。
- 逐文件复制时按文件数量更新安装进度条。
- 发生错误立即进入回滚阶段。
- 回滚时显示 `安装失败，正在恢复原版本`。
- 回滚成功后显示错误摘要和 `关闭` 按钮，不自动重启。
- 回滚失败时同时显示安装错误和回滚错误，保留备份目录路径供人工恢复。
- 安装成功后等待约 1 秒，再启动新版主程序并关闭更新器。

## 文件保护

继续保留：

- `user_config.yaml`
- `launcher_settings.json`
- `rename_log.csv`
- `organizer_run_log.json`
- `launcher_run_output.log`
- `update_check.log`

安装器仍执行 ZIP 路径安全检查，不允许绝对路径或 `..` 路径。现有文件替换前必须备份，异常时按相反顺序回滚。

## 错误处理

- 网络断开：显示 `下载中断，请检查网络后重试`。
- 无法获取总大小：使用不确定进度条，不将其视为错误。
- 用户停止：显示 `更新已停止，未修改任何程序文件`。
- SHA-256 不一致：删除 ZIP，显示 `更新包校验失败，请重新下载`。
- 无写入权限：在进入安装器前检测并提示。
- 更新器启动失败：主程序解除锁定并恢复可操作状态。
- 安装失败：更新器执行回滚，主程序不在失败状态下自动启动。

## 测试要求

### 下载逻辑

- 有 `Content-Length` 时正确报告百分比。
- 无 `Content-Length` 时报告累计大小但不计算百分比和剩余时间。
- 平均速度按累计字节和累计时间计算。
- 下载阶段取消会删除临时文件。
- 校验阶段取消会删除完整但未安装的 ZIP。
- SHA-256 失败会删除 ZIP。

### GUI 状态

- 自动检查和手动检查共用同一更新窗口。
- 更新中只存在一个更新窗口。
- 下载时主窗口被遮罩且不能操作。
- 停止后解除遮罩和操作锁。
- 准备安装后停止按钮禁用。
- 更新窗口在可取消阶段不能通过右上角关闭，只能点击停止。
- 各阶段的按钮、状态文案、进度条模式符合状态机。

### 更新器

- 安装成功按阶段报告进度并重启程序。
- 安装失败自动回滚。
- 回滚成功保留用户文件。
- 回滚失败显示备份目录。
- ZIP 路径穿越继续被拒绝。

### 发布验证

- 全量单元测试和语法检查通过。
- 重新构建主程序与 `updater.exe`。
- 用发布 ZIP 对临时旧版本执行真实升级。
- 验证用户配置和运行日志保持不变。
- 启动打包后的主程序和更新器窗口进行冒烟测试。

## 不在本次范围

- 不做断点续传。
- 不做后台静默更新。
- 不允许安装阶段取消。
- 不下载增量补丁，仍使用完整 ZIP。
- 不增加更新频道、测试版通道或自动定时安装。
