# Windows 文件整理助手现代化启动器 UI 设计

## 背景

现有 `launcher_gui.py` 是一个 Tkinter/ttk 图形启动器，负责选择 Python 命令、`file_helper.py`、待整理 root、`config.yaml`、运行模式和可选压缩，并通过 PowerShell 调用核心脚本。核心整理逻辑仍然在 `file_helper.py` 中，包括分类、数量识别、合并、重命名、压缩、撤销、确认和日志。

本次目标是把启动器升级为现代化 Windows 桌面界面，同时保持它只是启动器，不复制或改写文件整理业务逻辑。

## 设计目标

- 使用 `CustomTkinter` 重做启动器外观和交互。
- 保持默认预览优先，真实执行和撤销仍然需要 GUI 二次确认。
- 保留现有“保存设置 / 清空设置”能力。
- 增加轻量体验提示：当前模式、安全状态、路径是否存在、命令是否会带 `--yes`。
- 保持中文界面；CLI 参数、文件名和代码符号保持原样。

## 非目标

- 不重写 `file_helper.py`。
- 不把分类、合并、命名、压缩或撤销逻辑搬进 `launcher_gui.py`。
- 不在 GUI 内解析整理结果或替代 `rename_log.csv` / `organizer_run_log.json`。
- 不把启动器做成完整日志控制台或后台管理系统。
- 不改变 CLI 安全规则。

## 技术方向

推荐方案是用 `CustomTkinter` 替换现有 Tkinter/ttk 视觉层。`launcher_gui.py` 仍然保留现有职责：

- 选择路径。
- 校验路径存在性。
- 生成 PowerShell 命令。
- 复制命令。
- 弹窗确认真实执行或撤销。
- 确认后追加一次性 `--yes`。
- 启动 PowerShell。
- 保存和清空启动器设置。

如果引入 `CustomTkinter`，后续打包需要同步更新 PyInstaller 配置，确保 EXE 能带上 `customtkinter` 的资源文件。

## 主布局

采用“侧栏 + 工作区”布局。

左侧深色侧栏固定显示三个模式：

- 预览模式：`--dry-run`，默认选中，显示“不会修改文件”。
- 执行整理：`--apply`，显示“需要确认”。
- 撤销上次：`--undo-last`，显示“根据撤销日志回退”。

右侧浅色工作区包含：

- 顶部标题和当前模式标签。
- 工具路径区：Python 命令、`file_helper.py`、`config.yaml`。
- 任务路径区：要处理的 root 文件夹。
- 选项区：压缩最终文件夹、处理完成后打开结果目录。
- 命令预览区：显示将要运行的 PowerShell 命令。
- 底部动作区：生成命令、复制命令、在 PowerShell 中运行、保存设置、清空设置。

## 视觉规范

风格方向为“工具型工业清爽”：

- 深墨侧栏：用于模式导航和安全状态。
- 浅灰主背景：降低视觉负担。
- 白色输入卡片：承载路径和选项。
- 橙色主动作按钮：用于运行或关键动作。
- 绿色安全状态：用于 dry-run、安全提示和路径可用状态。
- 红/橙色危险提示：用于执行、撤销、路径缺失或错误。

卡片和按钮圆角控制在约 8px。界面应像稳定的 Windows 工具，而不是营销页面。

## 控件设计

路径输入：

- 每个路径行包含标签、输入框、选择按钮和状态文字。
- 状态文字只基于路径存在性，例如“已找到”“未选择”“路径不存在”。
- `config.yaml` 在撤销模式下不参与命令生成，可弱化或禁用。

模式切换：

- 用左侧大按钮替代当前 radio。
- 切换模式后立即更新说明、命令预览状态和可用选项。
- 撤销模式自动禁用压缩选项。

命令预览：

- 普通生成和复制命令不带 `--yes`。
- 只有点击运行，且用户确认执行或撤销后，才生成带 `--yes` 的一次性命令。
- 预览区需要明确提示“普通预览不带 --yes”。

设置：

- 保留保存设置和清空设置。
- 设置文件仍为 `launcher_settings.json`。
- 界面提示“保存设置只记录路径和启动选项，不保存客户文件内容”。

## 安全交互

默认模式仍为 dry-run。真实执行规则保持不变：

- `apply` 点击运行前弹出中文确认框。
- `undo-last` 点击运行前弹出中文确认框。
- 用户确认后才追加 `--yes` 并启动 PowerShell。
- 手动复制的普通命令不带 `--yes`。
- dry-run 不允许追加 `--yes`。
- `--archive` 只在 apply 模式可用。
- 撤销模式不带 `--config`、`--apply`、`--dry-run` 或 `--archive`。

GUI 只负责确认和命令构造，不改变 `file_helper.py` 的 CLI 校验。

## 错误处理

沿用当前保守错误处理：

- Python 命令为空时弹窗提示，不运行。
- `file_helper.py` 缺失时弹窗提示，不运行。
- root 文件夹缺失或不是目录时弹窗提示，不运行。
- 非撤销模式下，`config.yaml` 填写但不存在时弹窗提示，不运行。
- PowerShell 启动失败时弹窗提示。
- `launcher_settings.json` 损坏时弹窗提示，并回退默认设置。

新增状态提示不得从客户文件内容推断整理结果，只显示启动器本身能确定的信息。

## 测试计划

基础检查：

```powershell
python -m py_compile file_helper.py launcher_gui.py
```

GUI 手动验证：

- 启动源代码版 GUI。
- 验证默认进入预览模式。
- 验证三种模式生成的命令。
- 验证普通命令预览和复制不带 `--yes`。
- 验证执行和撤销确认后才追加 `--yes`。
- 验证撤销模式禁用压缩选项。
- 验证 `--archive` 只在执行模式出现。
- 验证保存设置和清空设置。
- 验证路径缺失、脚本缺失、root 缺失、config 缺失的错误提示。

如果需要更新发布文件夹：

- 同步新版 `launcher_gui.py`、`README.md`、`VERSION.txt` 等相邻文件。
- 如果引入 `CustomTkinter`，重建 foldered PyInstaller 发布包。
- 明确说明 EXE 是否重建，以及 `Windows文件整理助手-v1.1/` 是否已同步。
