# 扫描预览表格可展开列设计

## 背景

扫描预览表格包含较多列，部分内容超过默认列宽后无法直接看到完整文本。
当前 `原文件夹`、`目标名称` 和 `原因` 列允许自动拉伸，导致各列宽度不稳定，
同时长内容仍可能需要横向滚动才能查看。

## 目标

- 所有列默认使用稳定的固定宽度，不随窗口剩余空间自动拉伸。
- 用户点击任意单元格后，可以临时展开该单元格所在列。
- 展开宽度按该列所有数据行和表头中的最长文本计算。
- 多列可以同时保持展开。
- 再次点击已展开列中的任意单元格时，该列恢复默认宽度。
- 不修改扫描数据、分类、合并、重命名、执行、撤销或压缩逻辑。

## 默认列宽

| 列 | 默认宽度 |
| --- | ---: |
| 序号 | 48 |
| 原文件夹 | 210 |
| 识别日期 | 82 |
| 识别品类 | 118 |
| 命中关键词 | 130 |
| 单量 | 60 |
| 数量 | 60 |
| 动作 | 72 |
| 目标名称 | 220 |
| 状态 | 78 |
| 原因 | 240 |

所有列设置为 `stretch=False`，严格保持默认宽度。

## 交互规则

1. 用户单击表格中的任意数据单元格。
2. GUI 根据点击位置判断列标识。
3. 如果该列尚未展开：
   - 读取表头文字和该列所有数据行的文本。
   - 使用 Treeview 当前字体测量最长文本的像素宽度。
   - 增加适量左右内边距。
   - 展开宽度不得小于默认宽度，不得大于 `600px`。
4. 如果该列已经展开，则恢复该列的默认宽度。
5. 展开某列不会折叠其他列，多列可以同时展开。
6. 展开后使用表格底部现有横向滚动条查看超出可视区域的内容。
7. 点击表头、空白区域或分隔线不切换列宽。
8. 每次重新扫描前恢复全部默认列宽和折叠状态。

## 状态与组件边界

`launcher_gui.py` 负责：

- 保存当前已展开列集合。
- 处理 Treeview 单击事件。
- 测量内容宽度并更新列宽。
- 扫描前重置列宽。

`launcher_core.py` 负责保存默认列宽、最大宽度、内边距等纯配置，并提供展开宽度计算和列展开状态切换的纯函数，便于独立测试。

`file_helper.py` 不需要修改。预览行数据结构和核心整理流程保持不变。

## 宽度计算

使用 Tk 字体测量 API，而不是按字符数量估算，保证中文、英文和数字使用实际显示宽度。

计算规则：

```text
展开宽度 = min(600, max(默认宽度, 最长文本像素宽度 + 内边距))
```

建议内边距为 `24px`。如果最长文本仍超过 `600px`，用户通过横向滚动查看该列，
内容不会修改或截断。

## 测试与验证

新增可独立测试的列宽计算辅助逻辑，覆盖：

- 计算结果不小于默认宽度。
- 计算结果不超过 `600px`。
- 短文本保持默认宽度。
- 长文本按测量宽度加内边距。
- 多列展开状态互不影响。
- 再次点击同一列恢复默认宽度。
- 重新扫描恢复所有默认列宽。

GUI 手工验证：

1. 启动 `py launcher_gui.py`。
2. 扫描包含多行数据的测试目录。
3. 确认所有列初始宽度固定。
4. 点击“原因”列，确认按该列最长内容展开。
5. 点击“原文件夹”和“目标名称”，确认三列可同时展开。
6. 再次点击其中一列，确认只折叠该列。
7. 点击表头和空白区域，确认列宽不变化。
8. 再次扫描，确认所有列恢复默认宽度。
